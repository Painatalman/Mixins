    link-colors

    @mixin link-colors($link, $visited, $hover, $active, $focus) { ... }

    Description

    Link coloring mixin From the original source: The idea behind this mixin is that no defaults are declared in the arguments so that we are kind of forced in a friendly way to declare all 4 states of our links. The :focus and :active pseudo-classes are usually declared together. Feel free to separate them if you want to do so. One note about this mixin, is that it can be applied to any HTML element, not only links.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $link

    the default link color

    		Color—none
    $visited

    the color to be displayed on visited state

    		Color—none
    $hover

    the color to be displayed on hover

    		Color—none
    $active

    the color to be displayed when active

    		Color—none
    $focus

    the color to be displayed on focus (by default, it is the same as active)

    		Color—none

    Example

    a {
 @include link-colors(orange, blue, yellow, red);
}

    Links

    • http://codepen.io/ricardozea/pen/wMyZQe

    Author

    • Ricardo Zea

    reset-style-states

    @mixin reset-style-states($states) { ... }

    Description

    Reset style states Applies all rules inside a declaration to its selector and to all combinations of it concatenated with items in a set of pseudoclasses. This set pseudoclasses is provided through a parameter. For each state provided, the mixin joins the ampersand with the psedoclass indicator ':'. They are then concatenated to the main selector by commas

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $states

    the pseudoclasses the selector will be joined with in order to create the selector

    		Set of strings—none

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    /* the rules will expand to a declaration
 * containing '.hot-and-pink', '.hot-and-pink:hover',
 * '.hot-and-pink:active', '.hot-and-pink:focus' and '.hot-and-pink:visited'
 */
.hot-and-pink {
  @include reset-style-states {
    border-bottom: hotpink;
  }
}
