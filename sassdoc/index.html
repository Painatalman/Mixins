    link-colors

    @mixin link-colors() { ... }

    Description

    Link coloring mixin From the original source: The idea behind this mixin is that no defaults are declared in the arguments so that we are kind of forced in a friendly way to declare all 4 states of our links. The :focus and :active pseudo-classes are usually declared together. Feel free to separate them if you want to do so. One note about this mixin, is that it can be applied to any HTML element, not only links. @author Ricardo Zea @group elements @link http://codepen.io/ricardozea/pen/wMyZQe @param {color} $link the default link color @param {color} $visited the color to be displayed on visited state @param {color} $hover the color to be displayed on hover @param {color} $active the color to be displayed when active @param {color} $focus the color to be displayed on focus (by default, it is the same as active) @return {group of rules} Group of rules for coloring the linking element based on the provided colors @example scss - Link colors a { @include link-colors(orange, blue, yellow, red); }

    Parameters

    None.

    @function px2em($px, $default-font-size:) { ... }

    Description

    Pixels to Em, adapted from 2 authors Convert pixels to em. By default, the font size of the body is considered to be 16px

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $px

    The pixel value to be converted to em values

    		String—none
    $default-font-size:

    16px The body pixel value. By default, it is 14px

    		[type]—none

    Returns

    String —

    An em value, along with the unit
